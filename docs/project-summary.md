# Краткое описание проекта

Тема проекта: разработка консольного Java-приложения для управления пользователями с хранением данных в PostgreSQL.

Цель: закрепить практические навыки разработки корпоративного программного обеспечения на Java: разбиение на слои, работа с базой данных, JDBC, DTO, Maven и базовая безопасность.

## Функциональные возможности

1. Регистрация пользователя по email, паролю и описанию профиля.
2. Вход пользователя по email и паролю.
3. Поиск пользователя по идентификатору.
4. Обновление описания профиля.
5. Вывод всех пользователей.
6. Поиск пользователей по описанию профиля.

## Архитектурные решения

Приложение разделено на слои:

- `console` - взаимодействие с пользователем через терминал;
- `service` - бизнес-логика, валидация, проверка пароля;
- `repository` - доступ к данным;
- `dto` - безопасные объекты для вывода наружу;
- `security` - хеширование и проверка паролей;
- `config` - загрузка настроек приложения.

Пароль не передается в DTO и не выводится в консоль. В базе данных хранится только хеш, сформированный через PBKDF2.

## Хранение данных

Основной режим - PostgreSQL. Для быстрой демонстрации предусмотрен файловый режим, который использует тот же интерфейс репозитория и не требует изменения сервисного слоя.
